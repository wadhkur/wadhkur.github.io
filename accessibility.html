Accessibility commitments

Current implementation focus

Semantic structure

Pages use heading hierarchy, landmark elements, and descriptive link labeling.

Keyboard support

Primary interactions are reachable by keyboard with visible focus indicators.

Motion preferences

Animation behavior adapts when reduced-motion preferences are enabled.

Responsive readability

Typography and layout scale with fluid values across small phones through desktop displays.

Navigation consistency

Header, mobile navigation, and footer structure stay consistent across pages.

Feedback channel

Accessibility issues reported to the clinic team can be reviewed for practical remediation.

Review cycle

Accessibility QA flow

Baseline checks

Structure, focus behavior, and visual contrast are reviewed across templates.

Device-width verification

Layouts are validated at representative mobile, tablet, laptop, and desktop widths.

Issue remediation

Detected accessibility gaps are prioritized and addressed within the static-site design constraints.
